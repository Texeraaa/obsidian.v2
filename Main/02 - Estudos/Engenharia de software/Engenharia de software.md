___
Essa disciplina tem como objetivo geral apresentar conceitos de Engenharia de software e como aplicar esses conceitos em desenvolvimento de projeto
___
### Como surgiu?
Essa diciplina surgiu devido ao crescimento de complexidade dos softwares e enfrentar seus desafios de desenvolvimento

Esse termo surgiu em uma [[Crise do software|conferencia]] da OTAN sobre engenharia de software em 1968 na Alemanha

### Programação X Eng. de software

|               | Programação              | Engenharia de Software     |
| ------------- | ------------------------ | -------------------------- |
| **Escopo**    | Escrever código          | Ciclo de vida completo     |
| **Abordagem** | Tarefa pontual           | Visão holística            |
| Metodologia   | Ad Hoc                   | Agile, Waterfall, DevOps   |
| Colaborações  | individual/pequeno grupo | Equipes multidiciplinaries |
| Foco          | Problema técnico         | Problema técnico e humano  |
### Critérios de um bom software

- Qualidade: conformidade, confiabilidade, robustez e segurança
- Manutenibilidade: código limpo, modular e com design escalável, facilitando futuras manutenções
- Eficiencia: uso otimizado de memoria, CPU e tempo de resposta
- Usabilidade: UI/UX
- Custo-beneficio: entregar valor ao cliente durante o desenvolvimento

### Responsabilidades de um dev

- Entender o problema antes de escrever qualquer linha de código
- Planejar estimativa prazos e riscos
- Pensar em arquiteturas, modelagem, OO
- Comunicar bem com clientes e equipe
- Evoluir software de forma escalavel
